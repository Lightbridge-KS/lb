API Reference

File Structure

lb.core.fs.read_text_dir(dir_path)[source]

Read all text files in a directory and return their contents in a dictionary.

Parameters:

dir_path (str or Path) – Path to the directory containing text files.

Returns:

Dictionary with filenames as keys and file contents as values, sorted by filename.

Return type:

dict

Raises:

  • FileNotFoundError – If the specified directory does not exist.

  • NotADirectoryError – If the specified path is not a directory.

lb.core.fs.read_text_file(file_path)[source]

Read a text file and return its content as a string.

This function attempts to read a text file from the specified path using UTF-8 encoding.

Parameters:

file_path (str or Path) – The path to the text file to be read.

Returns:

The content of the text file as a string if successful. None: If the file is not found or an error occurs during reading.

Return type:

str

Raises:

  • FileNotFoundError – If the specified file path does not exist.

  • Exception – For other possible errors during file reading operations.

lb.core.fs.write_text_file(text, file_path)[source]

Write text content to a file at the specified path.

This function takes a string of text and writes it to a file, creating the file if it doesn’t exist or overwriting it if it does. The file is written using UTF-8 encoding.

Parameters:

  • text (str) – The text content to be written to the file.

  • file_path (str | Path) – The path where the file should be created/written to.

Raises:

Exception – Any exception that occurs during file operations will be caught and printed.

Example

>>> write_text_file("Hello World", "output.txt")
Text successfully written to output.txt

Reads the content of a .docx file and returns it as a string.

Parameters:

path (Path) – The path to the .docx file to be read

Returns:

The text content of the document with paragraphs joined together

Return type:

str

Raises:

  • docx.opc.exceptions.PackageNotFoundError – If the file is not found

  • docx.opc.exceptions.InvalidXmlError – If the file is not a valid .docx file

lb.docx.docx.read_docx_dir(directory: Path, recursive=True) → dict[source]

Reads all .docx files in a directory and returns a dictionary.

Parameters:

  • directory (Path) – Path to directory containing .docx files

  • recursive (bool, optional) – Whether to search subdirectories recursively. Defaults to True.

Returns:

Dictionary with filenames (without extension) as keys and document contents as values

{filename1: doc_content1, filename2: doc_content2, …}

Return type:

dict

lb.docx.docx.read_docx_dir_df(directory: Path, recursive=True) → DataFrame[source]

Reads all .docx files in a directory and returns a pandas DataFrame.

This function scans a directory for .docx files and creates a DataFrame containing the filename, full path, and content of each document.

Parameters:

  • directory (Path) – Path object pointing to the directory to scan

  • recursive (bool, optional) – Whether to scan subdirectories recursively. Defaults to True.

Returns:

DataFrame with columns:

  • filename: Name of the file without extension

  • path: Full path to the file

  • content: Text content of the document

Return type:

pd.DataFrame

Examples

>>> docs_df = read_docx_dir_df(Path("documents/"))
>>> print(docs_df.columns)
Index(['filename', 'path', 'content'], dtype='object')
